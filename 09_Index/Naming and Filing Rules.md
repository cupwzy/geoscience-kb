# Naming and Filing Rules

## File Names

- Concept notes: use stable English terms, for example `Elastic Impedance`.
- Project notes: use `Project - Area - Topic`, for example `Project - Block A - AVO Screening`.
- Paper notes: use `Author Year - Short Title`.
- Dataset notes: use `Dataset - Area - Object - Version`.
- Figure notes: use `Figure - Project - Message`.

## Where Things Go

- Temporary thought -> [[00_Inbox/Inbox]]
- Daily work log -> `00_Inbox/Daily`
- Domain knowledge -> matching MOC folder
- Active study or deliverable -> [[05_Projects/_Projects_MOC]]
- Literature card -> [[06_Papers_Zotero/_Papers_Zotero_MOC]]
- Reusable figure -> [[07_Figures/_Figures_MOC]]
- Dataset description -> [[08_Data/_Data_MOC]]

## Note Quality Levels

- `seed`: starter note or placeholder.
- `draft`: useful but not yet checked.
- `active`: currently trusted and maintained.
- `review`: needs validation before reuse.
- `archived`: preserved for history, not current guidance.

## Linking Rule

Prefer links that explain why a note matters:

- Concept -> Project: where the idea was used.
- Project -> Evidence: what supports the interpretation.
- Paper -> Concept: what knowledge was extracted.
- Dataset -> Experiment: where the data was consumed.
